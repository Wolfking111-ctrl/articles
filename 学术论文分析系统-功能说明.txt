一、系统概述
------------------------------------------------------------
本系统是一个基于 Dify 平台构建的学术论文智能分析工作流。
用户可以通过自然语言提问，系统自动结合本地知识库（行人重识别论文数据集）
和外部学术数据库（Semantic Scholar），对论文进行多维度分析，
并生成包含 ECharts 交互式图表的 HTML 可视化报告。

应用场景：学术研究、论文阅读、文献综述、实验对比分析


二、工作流整体流程
------------------------------------------------------------
用户输入（提问 + 可选上传PDF）
       ↓
[1] HTTP请求 → Semantic Scholar 学术API，获取相关论文及引用数据
       ↓
[2] 知识库检索 → 从本地论文数据集中检索相关内容（混合检索+Rerank）
       ↓
[3] 意图分类(LLM) → 判断用户意图：分析 / 对比 / 总结
       ↓
[4] 条件分支 → 根据意图分流到不同处理分支
       ↓
   ┌───────┬───────┬───────┐
   ↓       ↓       ↓       ↓
  分析    对比    总结    （默认:分析）
  分支    分支    分支
   ↓       ↓       ↓
[5]迭代  [6]LLM  [7]LLM
  逐段    对比    总结
  分析    分析    生成
   ↓
[8]变量聚合器（仅分析分支）
  合并: 迭代输出 + HTTP结果 + 知识库结果
   ↓       ↓       ↓
   └───────┴───────┘
           ↓
[9] 代码执行 → 生成 ECharts 图表 + HTML 可视化报告
           ↓
[10] 输出报告 → 展示完整的 HTML 分析报告


三、各节点详细说明（共 13 个节点）

------------------------------------------------------------
节点 1：开始 (start)
------------------------------------------------------------
类型：start
作用：工作流入口，接收用户输入
输入变量：
  - query（必填）：用户提问，如"分析这篇论文的创新点"
  - paper_file（可选）：支持上传 PDF 论文文件
涉及知识点：① 文件上传


------------------------------------------------------------
节点 2：学术检索 (http_s2)
------------------------------------------------------------
类型：http-request
作用：调用 Semantic Scholar 免费学术 API，搜索与用户问题相关的论文
API 地址：https://api.semanticscholar.org/graph/v1/paper/search
请求方式：GET
返回数据：论文标题、作者、年份、摘要、引用次数、URL
涉及知识点：② HTTP 请求
重要说明：Semantic Scholar 是免费 API，无需注册或 API Key


------------------------------------------------------------
节点 3：知识库检索 (knowledge)
------------------------------------------------------------
类型：knowledge-retrieval
作用：从本地知识库中检索与用户问题相关的论文内容
检索模式：混合检索（向量检索 + 关键词检索，权重各 50%）
Rerank 模型：qwen3-rerank（对检索结果重排序，提高准确性）
配置参数：
  - top_k: 5（返回前5个最相关片段）
  - score_threshold: 0.3（分数阈值，低于此值的结果被过滤）
涉及知识点：③ 知识库检索（RAG）


------------------------------------------------------------
节点 4：意图分类 (llm_intent)
------------------------------------------------------------
类型：llm
作用：分析用户输入，判断用户意图属于哪一类
输出分类：
  - "分析" → 进入深度分析分支（迭代逐段分析）
  - "对比" → 进入对比分析分支
  - "总结" → 进入快速总结分支
模型：qwen3-8b（temperature=0.1，低温度保证分类稳定）
涉及知识点：④ LLM


------------------------------------------------------------
节点 5：意图分流 (if_else)
------------------------------------------------------------
类型：if-else
作用：根据意图分类结果，将流程分流到3个不同的处理分支
分支条件：
  - 条件1（true）：包含"分析" → 进入迭代分析分支
  - 条件2：包含"对比" → 进入对比分析分支
  - 条件3：包含"总结" → 进入总结分支
涉及知识点：⑤ 条件分支


------------------------------------------------------------
节点 6：逐段深度分析 (iteration) + 段落分析 (llm_analyze)
------------------------------------------------------------
类型：iteration（外层容器）+ llm（内层分析节点）
作用：对知识库检索返回的每一段内容，逐个进行深度分析
迭代输入：知识库检索结果数组（knowledge.result）
迭代输出：每段内容的分析结果数组
分析维度：
  - 技术要点：算法、模型或技术细节
  - 创新贡献：体现的创新思想
  - 实验细节：实验设置、数据集、评估指标
  - 与其他工作关联：引用的相关工作
涉及知识点：⑥ 迭代
模型：qwen3-8b（temperature=0.3）


------------------------------------------------------------
节点 7：变量聚合器 (var_agg)
------------------------------------------------------------
类型：variable-aggregator
作用：合并多个数据源，为后续代码生成提供统一输入
聚合内容：
  - 知识库检索结果（knowledge.result）
  - Semantic Scholar API 返回结果（http_s2.body）
  - 迭代分析输出（iteration.output）
涉及知识点：⑦ 变量聚合器
重要说明：这是区分高分作业的关键技术点！
          迭代 + 变量聚合器的组合在 Dify 工作流中属于高级用法


------------------------------------------------------------
节点 8：论文总结 (llm_summary)
------------------------------------------------------------
类型：llm
作用：快速生成论文结构化总结（仅在"总结"分支执行）
输出结构：
  1. 研究背景与问题
  2. 核心方法
  3. 主要贡献
  4. 实验结论
  5. 局限性
模型：qwen3-8b（temperature=0.3）
涉及知识点：④ LLM


------------------------------------------------------------
节点 9：论文对比 (llm_compare)
------------------------------------------------------------
类型：llm
作用：对多篇论文进行多维度对比分析（仅在"对比"分支执行）
对比维度：
  1. 研究问题异同
  2. 方法对比（技术路线、模型架构）
  3. 数据集和评估基准
  4. 性能对比（Rank-1、mAP 等关键指标）
  5. 优缺点分析
  6. 发展趋势
模型：qwen3-8b（temperature=0.3）
涉及知识点：④ LLM


------------------------------------------------------------
节点 10：生成HTML报告 (code_html)
------------------------------------------------------------
类型：code
作用：Python 代码节点，处理所有分支的分析结果，生成完整的 HTML 可视化报告
处理逻辑：
  1. 判断哪个分支有输出（summary/compare/analysis）
  2. 解析 Semantic Scholar 数据，提取引用量等统计信息
  3. 生成 ECharts 柱状图配置（相关论文引用量对比）
  4. 组装完整的 HTML 页面（卡片统计 + 图表 + 分析详情）
报告内容：
  - 统计卡片：知识库片段数、相关论文数、总引用次数、分析模式
  - ECharts 柱状图：相关论文引用量对比
  - 分析详情：Markdown 格式的完整分析内容
涉及知识点：⑧ 代码执行 + ⑨ HTML 可视化（ECharts）


------------------------------------------------------------
节点 11：输出报告 (answer)
------------------------------------------------------------
类型：answer
作用：输出最终的分析报告，包含 HTML 内容和统计摘要
输出内容：分析模式 + 知识库片段数 + 相关论文数 + 完整 HTML 报告


四、环境变量说明
------------------------------------------------------------
系统配置了 3 个环境变量，方便调参：

1. S2_API_BASE
   值：https://api.semanticscholar.org/graph/v1
   作用：Semantic Scholar API 基础地址
   说明：一般不需要修改，除非 API 地址变更

2. SCORE_THRESHOLD
   值：0.3
   作用：知识库检索的分数阈值
   说明：值越低，召回结果越多但可能不相关；值越高，结果越精确但可能太少

3. TOP_K
   值：5
   作用：知识库检索返回的最大结果数
   说明：影响迭代分析的数量，建议 3-8 之间


五、你需要修改的部件（重要！）
------------------------------------------------------------

【必须修改】

1. 知识库数据集 ID
   位置：节点"知识库检索" → dataset_ids
   当前值：9ecc7ab3-a8d9-4656-99f5-796e3906c09b
   修改为：你的实际知识库数据集 ID
   查找方法：Dify 控制台 → 知识库 → 选择你的数据集 → URL 中的 UUID

2. 模型提供商名称
   位置：所有 LLM 节点（4个）→ model.provider
   当前值：tongyi
   修改为：你在 Dify 中配置的实际提供商名称
   常见值：tongyi / siliconflow / openai_api_compatible
   注意：同时检查 model.name 是否在你的提供商中可用

3. Rerank 模型
   位置：节点"知识库检索" → multiple_retrieval_config.reranking_model
   当前配置：provider: tongyi, model: qwen3-rerank
   修改为：你实际可用的 Rerank 模型
   如果不需要 Rerank，可以将 reranking_enable 改为 false

【建议检查】

4. Semantic Scholar API 连通性
   该 API 免费无需 Key，但需要确认你的 Dify 服务器能访问外网
   测试方法：在浏览器访问 https://api.semanticscholar.org/graph/v1/paper/search?query=person+reid&limit=1

5. 代码节点中的 Python 依赖
   代码节点使用了 json 模块（Python 内置），不需要额外安装依赖

6. ECharts CDN
   HTML 报告引用了 CDN：
   https://cdn.jsdelivr.net/npm/echarts@5.5.0/dist/echarts.min.js
   需要确认网络能访问此 CDN

【可选调整】

7. 检索参数优化
   环境变量 SCORE_THRESHOLD 和 TOP_K 可以根据实际效果调整

8. LLM 温度参数
   意图分类：temperature=0.1（建议保持低温度）
   分析/总结/对比：temperature=0.3（可根据需要调整）

9. 添加更多知识库
   如果有多份论文数据，可以在知识库检索节点中配置多个 dataset_id


六、导入步骤
------------------------------------------------------------
1. 打开 Dify 平台 → 点击"创建应用" → 选择"导入 DSL 文件"
2. 上传 academic-paper-analysis.yml
3. 导入后会自动创建工作流，但需要手动配置：
   a. 选择知识库（点击知识库检索节点 → 选择你的数据集）
   b. 检查模型配置（每个 LLM 节点 → 选择正确的模型提供商和模型）
   c. 检查 Rerank 配置
4. 点击"发布"按钮
5. 在"预览"或"API 访问"中测试

八、演示建议
------------------------------------------------------------
高分演示技巧：
1. 先展示"总结"模式：输入"总结这篇论文"，快速得到结构化总结
2. 再展示"分析"模式：输入"分析这篇论文的方法创新点"，展示迭代
